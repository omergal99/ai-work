# AI Documentation Standards

## Purpose
These standards define how .ai content should be written so that humans and agents can both use it effectively.

## Core Rules
- Prefer short sections.
- Use clear headings and IDs.
- Keep content structured and scannable.
- Use telegraphic English where appropriate.
- Avoid filler and anecdotal prose.
- Keep one idea per section.
- Make completion criteria explicit.

## Required Structure
1. Executive Summary
2. Why It Matters
3. Mental Model
4. Architecture
5. Best Practices
6. Common Mistakes
7. Implementation Examples
8. Integration into the Repository
9. Integration with AI Agents
10. Recommended Directory Structure
11. Files That Should Exist
12. Templates
13. Checklists
14. Future Trends
15. References

## Maintenance Rule
When content changes, update the related documents and the main index.
