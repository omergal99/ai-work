DOC_ID: SKILL_BUILDER_01
Keywords: builder, explorer, workflow, agents, evals

PURPOSE
Create a developer-facing builder explorer for GenAI flows.

WHEN TO USE
Use when the task is to design, structure, or explain a multi-agent workflow for developers.

STEPS
1. Understand the product goal.
2. Map the user journey and core tasks.
3. Identify modules, agents, tools, and prompts.
4. Define state schema and handoff rules.
5. Add eval cases for capability and regression.
6. Produce a concise builder blueprint.

COMPLETION CRITERIA
- Every major step has a clear owner.
- State is explicit and centralized.
- Evals exist for critical flows.
- Output is structured and developer-ready.

RULES
- Prefer small, focused agents.
- Use explicit handoffs.
- Keep prompts deterministic.
- Do not rely on long chat history for state.
- Favor testable outcomes over vague prose.
