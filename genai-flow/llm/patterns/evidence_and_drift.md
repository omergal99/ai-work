# Evidence and Drift

## Purpose
Make sure outputs are not only fluent, but also traceable and stable over time.

## Core rules
- every important claim should be linked to evidence
- every answer should be checked against current context before acceptance
- every repeated workflow should be evaluated for drift over time

## Drift signals
- output quality changes without a clear reason
- retrieval quality changes unexpectedly
- prompt or state assumptions start to diverge from reality

## Recommended practice
1. Compare current outputs to previous ones for similar tasks.
2. Flag any disagreement that is not justified by new evidence.
3. Record the drift and adjust the workflow, prompt, or retrieval strategy.

## Benefits
- improves reliability
- makes review easier
- keeps agents aligned with the latest reality
