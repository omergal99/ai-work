# Configuration Layer

This directory is the first stop for any agent or contributor entering this repository.
It contains the shared state, operating rules, and adoption guidance that define how work should happen here.

## What to read first
1. Read this file.
2. Review the shared state in brain_state.json.
3. Read context_lake_blueprint.md for context and memory rules.
4. Review jcode_swarm_rules.md for execution and handoff conventions.
5. Use adoption_playbook.md for rollout and maturity guidance.

## Core files
- brain_state.json — shared manifest and state pointers
- context_lake_blueprint.md — context storage, memory, and retrieval rules
- jcode_swarm_rules.md — swarm execution and local workflow conventions
- adoption_playbook.md — adoption roadmap and rollout guidance

## Operating principle
Keep this layer small, clear, and stable. New agents should be able to understand the repository by reading this directory first rather than discovering rules from scattered files.
