# AI Engineering Learning Roadmap

## Executive Summary
This roadmap is for building practical AI engineering capability in a repository-oriented environment. The order matters: start building early, learn fundamentals as needed, and keep the system observable and eval-driven.

## Core Sequence
1. LLM APIs
   - Model types
   - Structured outputs
   - Prompt caching
   - Multimodal models

2. Model Adaptation
   - Prompt engineering
   - Tool use
   - Fine-tuning

3. Retrieval and Storage
   - Vector databases
   - Graph databases
   - Hybrid retrieval

4. RAG and Agentic RAG
   - Data preparation
   - Retrieval and generation
   - Reranking
   - MCP
   - Orchestration

5. AI Agents
   - Design patterns
   - Multi-agent systems
   - Memory
   - Human-in-the-loop and human-on-the-loop

6. Infrastructure
   - Kubernetes
   - Cloud services
   - CI/CD
   - Model routing
   - LLM deployment

7. Observability and Evaluation
   - Instrumentation
   - Observability platforms
   - Evaluation techniques
   - Regression and capability evals

8. Security and Governance
   - Guardrails
   - Testing
   - Ethical considerations

## Repository Usage
This repository should evolve in phases:
- Phase 1: local documentation, memory, and templates
- Phase 2: structured retrieval and MCP integration
- Phase 3: multi-agent orchestration and evals
- Phase 4: production-grade observability and cost control
