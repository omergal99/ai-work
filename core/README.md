# Core Operating Layer

This directory consolidates the main operational structure of the repository.
It contains the shared agents, protocols, prompts, skills, templates, and operational standards that support the rest of the workspace.

## What lives here
- agents/ — agent roles and collaboration conventions
- protocols/ — contracts, standards, and operating rules
- prompts/ — reusable prompts
- skills/ — reusable skill instructions
- templates/ — reusable templates for tasks, evals, and state manifests
- ops/ — operational discipline and history
- mcp/ — MCP configuration and registry
- observability/ — tracing and monitoring guidance
- deployment/ — deployment guidance
- orchestration/ — multi-agent execution and coordination models
- control-plane/ — control plane concepts and schemas
- repository-intelligence/ — repository intelligence guidance
- context/ — context engineering guidance
- evals/ — evaluation guidance and example cases

## Why this layer exists
This layer keeps the repository practical and consistent. Instead of scattering core operating knowledge across the root, it centralizes it in one place that is easier to navigate and easier for agents to reason about.
